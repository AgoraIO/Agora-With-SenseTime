详细描述

基于GPU（OpenGL）的YUV/RGB颜色转换接口， 支持rgba图像和nv21、nv12以及yuv图像之间的转换

注意
本头文件中的所有接口都需要在GL Context中调用， API不保证线程安全.多线程调用时,需要确保安全调用

一般调用步骤：创建句柄->准备输入、输出资源->调用转换接口->销毁句柄

st_handle_t handle = NULL;

st_mobile_color_convert_create(&handle);

st_mobile_nv12_buffer_to_rgba_tex(handle, 1280, 720, ST_CLOCKWISE_ROTATE_0, false, p_buffer, tex_id);

st_mobile_color_convert_destroy(handle);

handle = NULL;

ST_SDK_API st_result_t st_mobile_nv21_buffer_to_rgba_tex ( st_handle_t  handle,
int  width,

int  tex_out 
)

对输入的nv21格式的buffer转换成RGBA格式，并输出到texId对应的OpenGL纹理中，需要在OpenGL Context中调用

参数
[in]handle已初始化的颜色格式转换句柄
[in]width待转换图像的宽度
[in]height待转换图像的高度
[in]orientation图像朝向，根据传入图像旋转角度，将图像转正。如果旋转角度为90或270，tex_out的宽高需要与buffer的宽高对调。
[in]horizontal_mirror是否需要水平镜像，true - 水平镜像，false - 垂直镜像
[in]p_bufferNV21格式的图像buffer，需要预先分配空间（字节数：width * height * 3 / 2)
[out]tex_outRGBA格式输出纹理，需要在调用层预先创建
返回
成功返回ST_OK, 错误则返回错误码,错误码定义在st_mobile_common.h中,如ST_E_FAIL等
注意
需要在OpenGL Context中调用
