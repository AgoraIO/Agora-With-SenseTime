详细描述

提供授权文件激活功能

一般调用步骤：验证现有激活码->如果验证失败，生成新的激活码

if(ST_OK != st_mobile_check_activecode(...)) { 

st_mobile_generate_activecode(...)

Save New Active Code to disk

}

◆ st_mobile_generate_activecode()

ST_SDK_API st_result_t st_mobile_generate_activecode ( const char *  license_path,
char *  activation_code,
int *  activation_code_len 
)

根据授权文件生成激活码, 在使用新的license文件时使用. Android建议使用st_mobile_generate_activecode_from_buffer

参数
[in]license_pathlicense文件路径
[out]activation_code返回当前设备的激活码,由用户分配内存, 通常建议分配1024字节，如果使用动态在线鉴权，需要分配10240字节
[in,out]activation_code_len输入为active_code的内存大小, 返回当前设备的激活码字节长度（包含激活码字符串结束符'\0'）
返回
正常返回ST_OK,否则返回错误类型
